# ogori-games — 誰が払う?決めるゲーム集

大人数(2〜12人)で外食・外出したときに「誰がお金を払うか」を、
その場で盛り上がりながら公平に決めるためのスマホ向けWebアプリです。

1台のスマホを参加者間で回して使うことを想定しています。

## 対象ユーザー

- 大学生グループなど、2〜12人程度で出かけて割り勘/おごりを決めたい人
- スマホ1台をみんなで回して遊ぶシチュエーション

## 起動方法

ビルド不要・外部通信なし・外部ライブラリなしのVanilla HTML/CSS/JavaScriptです。

1. `src/index.html` をブラウザ(スマホ・PCどちらでも可)で直接開くだけで動作します。
2. ローカルサーバー経由で確認したい場合は、`src` ディレクトリで以下を実行してください。

   ```
   cd src
   python3 -m http.server 8000
   ```

   その後ブラウザで `http://localhost:8000/` を開いてください。

## 遊び方(共通の流れ)

1. **参加者入力画面**: 名前を入力してEnterまたは「追加」ボタンで参加者を追加します(2〜12人、削除も可能)。2人以上になると「ゲーム選択へ」ボタンが押せるようになります。
2. **ゲーム選択画面**: 4種類のゲームからカードをタップして選びます。「参加者を変更」で参加者入力画面に戻れます。
3. **各ゲーム画面**: ゲームごとの演出に従って全員が順番に操作します。
4. **結果画面**: 支払者の名前が大きく表示されます(🎉演出つき)。「もう一回」で同じゲームを最初からやり直せます。「ゲーム選択に戻る」で別のゲームを選び直せます。

全ゲーム共通で、順番制の場面では「○○さんの番」という表示のあとに操作するため、
スマホを渡すタイミングがわかりやすくなっています。

## ゲームのルール

### 🎡 男気ルーレット

- 参加者の名前が並んだルーレットを「回す」ボタンで回転させます。
- 3〜5秒かけて徐々に減速して停止し、矢印が指した人が支払者になります。

### 🃏 ハイ&ロー(カード引き)

- 52枚のトランプ(重複なし)から、参加者が1人ずつ順番にカードを1枚引きます。
- 「○○さんの番」の表示のあと、裏向きのカードをタップするとめくれます。
- 全員引き終わったら、一番弱いカードの人が支払者になります。
  - 強さは数値のみで比較します(2が最弱、Aが最強)。スート(マーク)は勝敗に関係しません。
- 最下位が同じ数値で複数いる場合は、該当者のみで山札をリセットして引き直します。

### 🎲 チンチロ

- 各自「振る」ボタンをタップして、サイコロ3つを振ります(1人1回、振り直しなし)。
- 出た目から役を判定し、以下の強さ順で比較します(左が最強)。

  ピンゾロ(1,1,1) > ゾロ目(同じ目3つ。数字が大きいほど強い) > シゴロ(4,5,6) >
  通常の目(2つ同じ+残り1つ。残りの数字が大きいほど強い) > 目無し(役なし) > ヒフミ(1,2,3)

- 全員振り終わったら、一番弱い役の人が支払者になります。
- 最下位の役が完全に同格の人が複数いる場合は、該当者のみで振り直します。

### 🪜 あみだくじ

- 参加者の人数分の縦線が引かれたあみだくじが表示されます。横線はランダムに生成されます。
- 各自、上部の空いている枠を順番にタップして選びます。
- 全員選び終わったら「結果を見る」ボタンで、線をたどるアニメーションが1人ずつ流れます。
- 下端のうち1箇所だけが「💸支払い」で、そこに到達した人が支払者になります。

## 参加人数

- 2人〜12人まで対応しています(参加者入力画面で追加・削除できます)。
- 同じ名前を複数追加した場合は「名前(2)」のように自動で連番が付きます。

## 技術スタック

- HTML / CSS / JavaScript(Vanilla)
- ビルドツール・外部ライブラリ・外部通信・CDN読み込みは一切なし
- スマホ縦画面(375px幅)を基準にしたレスポンシブデザイン

## 今回のスコープ外(やらないこと)

- 傾斜配分・割り勘の金額計算(全額おごりの1人を決めるだけです)
- 複数端末間の通信・オンライン対戦
- 結果の保存・履歴機能
- ネイティブアプリ化・PWA対応
