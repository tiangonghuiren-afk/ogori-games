# プロジェクトビジョン — ogori-games(誰が払う?決めるゲーム集)

- 目的: 大学生などのグループが外食・外出時に「誰がお金を払うか」を、その場で盛り上がりながら公平に決められるようにする
- 対象ユーザー: 大人数(2〜12人程度)で出かける大学生グループ。1台のスマホを回して使う想定
- 成功指標: 参加者の名前を入力してゲームを選ぶだけで、演出込みで支払者が1人決まり、「もう一回」ですぐ遊び直せる

## 必須機能

1. 参加者の名前入力(2〜12人、追加・削除可能)
2. ゲーム選択画面(4種類から選ぶ)
3. ゲーム4種:
   - **チンチロ**: 各自お椀にサイコロ3つを振る。1人3回まで振れ、役/目が出たらそこで確定、3回とも目無しなら目無しで確定。役の弱い人が支払い(引き分けは該当者で振り直し)
   - **男気ルーレット**: 参加者名のルーレットを回し、止まった人が支払い
   - **ハイ&ロー(カード引き)**: 各自カードを1枚引き、一番弱いカードの人が支払い(同点は引き直し)。カードは下からゆっくり絞って開く演出
   - **あみだくじ**: 下端の1箇所を「ハズレ(爆弾💣)」として最初から明示。上の枠を選び、「結果を見る」で横線が出現し線をたどるアニメーションで爆弾に着いた人が支払い
4. 結果画面(支払者をドーンと表示、「もう一回」「ゲーム選択に戻る」ボタン)

## 演出・効果音(スコープに追加)

- 全ゲームに効果音を付ける。音は外部ファイルを使わず WebAudio API で合成する(外部通信・外部ライブラリ禁止の方針を維持)
- ルーレット: 回転中「がらがら」という連続クリック音(減速に同期して間隔が伸びる)
- ハイ&ロー: カードをめくる「シャカ」というスライド音。演出は下から少しゆっくり絞って開く
- チンチロ: サイコロがお椀に落ちる「チリンチリン」という金属的な着地音
- あみだくじ: 線をたどる「トゥルッ」というスライド音

## 今回やらないこと

- BGM(効果音のみ対応。ループ音楽は入れない)
- 傾斜配分・割り勘計算(金額入力なし。全額おごり1人を決めるだけ)
- 複数端末間の通信・オンライン対戦
- 結果の保存・履歴機能
- ネイティブアプリ化・PWA対応

## 技術スタック

- HTML / CSS / JavaScript(Vanilla、ビルドツールなし)
- 外部ライブラリ・外部通信なし。ブラウザで `index.html` を開くだけで動作
- スマホ縦画面ファーストのレスポンシブデザイン

## 決定事項

- 支払者は常に1人(ゲームで負けた/当たった人)
- 乱数はゲームごとに公平(全員同条件)であること
- 1台のスマホを回して使う前提のUI(「○○さんの番」表示で受け渡しを促す)
