# プロジェクトビジョン — ogori-games(誰が払う?決めるゲーム集)

- 目的: 大学生などのグループが外食・外出時に「誰がお金を払うか」を、その場で盛り上がりながら公平に決められるようにする
- 対象ユーザー: 大人数(2〜12人程度)で出かける大学生グループ。1台のスマホを回して使う想定
- 成功指標: 参加者の名前を入力してゲームを選ぶだけで、演出込みで支払者が1人決まり、「もう一回」ですぐ遊び直せる

## 必須機能

1. 参加者の名前入力(2〜12人、追加・削除可能)
2. ゲーム選択画面(4種類から選ぶ)
3. ゲーム4種:
   - **チンチロ**: 各自3つのサイコロを振り、役の弱い人が支払い(引き分けは該当者で振り直し)
   - **男気ルーレット**: 参加者名のルーレットを回し、止まった人が支払い
   - **ハイ&ロー(カード引き)**: 各自カードを1枚引き、一番弱いカードの人が支払い(同点は引き直し)
   - **あみだくじ**: 上の枠を選ぶと線をたどるアニメーションで「支払い」に当たった人が決まる
4. 結果画面(支払者をドーンと表示、「もう一回」「ゲーム選択に戻る」ボタン)

## 今回やらないこと

- 傾斜配分・割り勘計算(金額入力なし。全額おごり1人を決めるだけ)
- 複数端末間の通信・オンライン対戦
- 結果の保存・履歴機能
- ネイティブアプリ化・PWA対応

## 技術スタック

- HTML / CSS / JavaScript(Vanilla、ビルドツールなし)
- 外部ライブラリ・外部通信なし。ブラウザで `index.html` を開くだけで動作
- スマホ縦画面ファーストのレスポンシブデザイン

## 決定事項

- 支払者は常に1人(ゲームで負けた/当たった人)
- 乱数はゲームごとに公平(全員同条件)であること
- 1台のスマホを回して使う前提のUI(「○○さんの番」表示で受け渡しを促す)
